# TNJIndex · 忠！诚！猫鼠队！

本文档面向 **项目作者与 Cursor / AI 助手**：建立共同上下文，避免需求与实现口径漂移。对外说明以根目录 `README.md` 为准（若与本文冲突，以 `docs/requirements/mvp_requirement.md` 为需求真源）。

## 定位

**TNJIndex**（中文站名：**忠！诚！猫鼠队！**）是一个 **《猫和老鼠》相关梗图 / 截图的索引与检索站**：用主题或标签把「和讨论语境迷之契合」的画面找出来，方便查询与二创参考。

## 动机

在 B 站、小红书、贴吧等社区里，常见一种现象：某个主题帖下，会有人贴出《猫和老鼠》里 **和主题并不字面相同、但气氛或构图莫名合拍** 的截图，节目效果很好。零散帖子不易复用，因此希望有一个 **可检索的集中索引**，把这类素材从「刷到算赚到」变成「按意图查找」。

## 目标用户与典型用途

- **社区发帖 / 回帖**：按当下话题快速找一张「对味」的截图。
- **二创与整理**：按主题、情绪、构图倾向浏览或定位素材。

（具体用户故事与验收以 MVP 文档为准。）

## 核心能力（产品层）

1. **自然语言查询**：用户用口语化描述想要的氛围或主题，系统返回相关梗图条目。
2. **结构化检索**：支持按主题、标签、描述等维度筛选或组合查询。

实现形态（是否登录、分页、排序规则等）不在此展开，见需求与阶段文档。

## 范围与非盈利

- 个人项目，**非盈利**；功能与数据规模以 MVP 与后续 Phase 为界。
- **明确不做**、边界条件见 [`requirements/mvp_requirement.md`](requirements/mvp_requirement.md)。

## 实现愿景（摘要）

整体路径可概括为三步（**细节与选型见架构与计划文档，此处仅对齐想象**）：

1. **采集**：积累大量《猫和老鼠》相关截图 / 梗图（来源与规模由后续数据方案约束）。
2. **标注与索引**：用 AI 辅助生成可被检索的文本信息（如标签、描述、与主题的关联说明等）；需统筹 **质量、一致性与调用成本**。
3. **检索**：在结构化字段上可做过滤与排序；在语义上与自然语言对齐时，可采用 **向量检索 + 重排** 或同类现成能力，具体以技术选型为准（不必在 intro 层绑定某一厂商或框架）。
