# 术语表（Glossary）

面向 **项目作者与 Cursor / AI 助手**：统一产品与实现用语，减少需求、代码与文档之间的歧义。

## 文档说明

- **维护方式**：新增或改名术语时，优先更新本表，再同步改 `intro_to_tnjinex.md`、`docs/mvp/` 与架构文档中的表述。
- **冲突处理**：若与项目其他文档不一致，以 **MVP 需求真源** 为准（当前仓库内见 [`mvp/`](mvp/) 下承担用户故事与验收说明的文档；若日后迁移路径，以该目录或替代文档中的明示为准）。
- **表格列**：**中文术语** | **English** | **定义** | **备注 / 易混**  
  **English** 列在架构与 API 命名定稿前统一填 `TBD`；定稿后改为具体单词并保持与代码一致。

## 核心业务概念

| 中文术语 | English | 定义 | 备注 / 易混 |
|----------|---------|------|-------------|
| 素材 | TBD | 系统内 **最小可检索、可展示单位**，对应一张（或一条记录所指向的）《猫和老鼠》相关画面内容及其元数据。 | 需求与 UI 文案中优先使用「素材」，避免与「数据库行」口语混用时不加说明。 |
| 截图 | TBD | 从成片截取的一帧或经裁切的静态图像，作为素材的图像载体。 | |
| 梗图 | TBD | 在社区语境中被当作 meme 使用的截图；**产品语义上常与「截图」混称**，强调传播与「对味」而非技术格式。 | 与「截图」在数据模型上通常对应同一类 **素材**；若需区分「是否经过再创作」由 MVP 约定。 |
| 主题 | TBD | 偏宏观、贴近讨论语境的话题维度（例如用户发帖时心里想的大致方向）。 | 与 **标签** 的层级关系、是否允许多值以 MVP 为准。 |
| 标签 | TBD | 结构化、可用于筛选与聚合的短词或短语，挂在素材上供查询。 | 与 **主题** 是否拆分、是否共用词表以 MVP 为准。 |
| 描述 | TBD | 以自然语言概括画面内容、氛围或与语境的关联，用于展示与检索。 | 可由模型或人工产生；是否多语言、长度上限由数据方案约定。 |
| 自然语言查询 | TBD | 用户输入自由文本，系统返回相关 **素材** 列表的查询方式。 | 与 **结构化检索** 互补；是否混合为同一搜索框由产品设计决定。 |
| 结构化检索 | TBD | 按 **主题**、**标签**、**描述** 等明确字段或组合条件筛选 **素材**。 | 含精确匹配、前缀、多选等形态，具体以 MVP 为准。 |

| 中文术语 | English | 定义 | 备注 / 易混 |

## 检索与语义（技术—产品）

| 中文术语 | English | 定义 | 备注 / 易混 |
|----------|---------|------|-------------|
| 向量检索 | TBD | 将文本或图像等映射为向量，在向量空间近似最近邻以召回候选 **素材**。 | 常与 **嵌入** 配合使用。 |
| 嵌入 | TBD | 将非结构化输入映射为固定维度向量的表示；模型与维度由选型决定。 | 亦作 embedding；对外文案可用「语义向量」等说法。 |
| 重排 | TBD | 在召回候选集之后，用更精细的模型或规则对结果排序以提升相关性。 | 可选阶段；是否采用以成本与效果权衡为准。 |
| RAG | RAG（可保留缩写） | 检索增强生成：为大模型检索外部片段再生成回答的范式。 | 本项目核心是 **素材检索与展示**；若仅检索库内 **素材** 而无「生成式回答」，则严格说不等于 RAG。若产品形态包含「解释为何匹配」等生成内容，再在本行补充界定。 |
